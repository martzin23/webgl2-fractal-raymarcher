\section{Funkcije udaljenosti}
\label{podpog:funkcije_udaljenosti}

Postoji više načina na koje se može zapisati neko tijelo. U računalnoj grafici se najčešće trodimenzionalni modeli opisuju mrežom trokuta. Trokuti su praktični jer imaju jednostavnu strukturu i mogu se efikasno iscrtati. Unatoč toga, postoje nedostatci takvim pristupom. Da bi se postigla zakrivljena površina, potrebno je izgraditi površinu s velikim brojem trokuta. Time površina nikad neće biti uistinu zakrivljena, nego će se sastojati od diskretnih ravnih dijelova. Neke strukture (kao na primjer spužvaste strukture) zahtijevaju toliko veliki broj trokuta da se treba napraviti kompromis brisanjem detalja strukture radi optimizacije. Detalji se općenito dodaju modelima koristeći teksture, no u tim slućajevima stvarna geometrija još uvijek ostaje pojednostavljena.

Način prikaza tijela koje će se razmatrati u ovom radu jer matematički opis. Specifično, ono što ovaj rad koristi su funkcije udaljenosti (\emph{eng. Signed Distance Function, SDF}). Oblik funkcije je sljedeći: $f(p)$. Gdje je $p$ točka u prostoru, a rezultat funkcije $f(p)$ je udaljenost te točke od površine tijela koju funkcija opisuje. Dimenzija prostora u kojem se treba nalaziti ulazna točka određuje funkcija. Primjer jedne takve funkcije za opis sfere je navedena ispod u jednađbi \eqref{eq:sdf}.

\begin{equation}
	\label{eq:sdf}
	f(p)_{\text{sfera}} = |centar - p| - radius
\end{equation}

Slika \ref{slk:sdf} skicira vrijednosti funkcije udaljenosti pravokutnika. Crte na slici prikazuju gdje su vrijednosti funkcije jednake (mogu se zamisliti kao izohipse na topografksoj karti). Bijela crta označuje gdje je vrijednost udaljenosti nula, odnosno površina tijela. Osim vraćanja vrijednost udaljenosti, drugo važno svojstvo funkcije je predznak rezultata. Vrijednost je pozivitna kada je točka izvan tijela (slika \ref{slk:sdf}, narančasti dio), a negativna kada je točka unutar tijela (slika \ref{slk:sdf}, plavi dio). Koristeći navedena svojstva je moguće opisati neko trodimenzionalno tijelo. Nadalje, to je i dovoljno za izgraditi prikaz tog tijela.

\section{Fraktali}
\label{podpog:fraktali}

Fraktali su oblici sa svojstvom samoponavljanja. Odnosno, postoji uzorak (\emph{pattern}) ili neko svojstvo koji se ponavlja po modelu, na raznim mjestima i raznim veličinama. Ponavljanje tog uzorka može ići do beskonačnosti, što znaći da je fraktal u teoriji ima beskonačno detalja. Takve fraktalne strukture se mogu pronaći u prirodi, kao na primjer stabla, lišće (slika \ref{slk:fern}) i reljef površine Zemlje.

\begin{figure}[htb]
	\centering
	\includegraphics[width=0.38\linewidth]{images/fern.jpg}
	\caption{Fraktalna struktura lista paprata \cite{fern}}
	\label{slk:fern}
\end{figure}

Najčešće se koristi matematički zapis za opis jednostavnih oblika kao ravnine, sfere ili trokuta. Ako pokušavamo matematički opisati neki kompleksniji oblik kao na primjer zec, postane očito da će to biti teško. Može se pretpostaviti da je kompleksnost formule raste s kompleksnosti oblika. No, to nije uvijek slučaj. Ispada da jednostavna formula može napraviti vrlo detaljan oblik, a dobar primjer toga su fraktali. Ovdje dođu funkcije daljine do koristi jer se mogu iskoristiti za opis fraktala.

\section{Algoritam iscrtavanja}
\label{podpog:algoritam_iscrtavanja}

Cilj je postići prikaz fraktala. Potreban je algoritam koji uzima opis tijela u obliku funkcije u daljenosti i iscrtava trodimenzionalni prikaz tog tijela na ekranu. Koristiti će se metoda praćenja sfere, što je podvrsta metode koračenja zrake, što je još podvrsta metode praćenja zrake. Metoda praćenja zrake (\emph{engl. Ray Tracing}) određuje boju svakog piksela pomoću zrake koja počinje u očištu kamere i prolazi kroz taj piksel. Metoda koračenja (\emph{engl. Ray Marching}) zrake koristi definiranu zraku tako da postepeno korači u smjeru zrake tijekom računanja. Praćenje sfere (\emph{engl. Sphere Tracing}) dalje definira koračenje tako da je duljina svakog koraka određena na temelju funkcije udaljenosti. Prikazivanjem svakog skoka i pripadajuće vrijednosti udaljenosti se dobiva slika sfera (ili kružnica ako se gleda u 2D prostoru), što se može vidjeti na slici \ref{slk:spheretracing}.

\begin{figure}[htb]
	\centering
	\includegraphics[width=0.5\linewidth]{images/spheretracing.png}
	\caption{Koraci zrake u praćenju sfere}
	\label{slk:spheretracing}
\end{figure}

Ispod je naveden pseudo kôd (Algoritam \ref{alg:spheretracing}) koji prikazuje postupak metode praćenja sfere. U svakoj iteraciji petlje se računa udaljenost pomoću funkcije udaljenosti i obavi se korak duljine te udaljenosti u smjeru zrake. Petlja završava kada je zraka dotaknula tijelo ili kada prođe granicu maksimalnog broja koraka. Mora se zadati granica koraka zbog slućaja kada zraka ne udari tijelo, nego nastavi koračiti u beskonačnost. Zbog nepreciznosti brojeva s pomičnim zarezom može doći do slučaja da zraka nikada ne dođe do udaljenosti nula od tijela. Pritom se za uvjet dodira dodaje varijabla \emph{epsilon}, koja ima vrijednost vrlo malog broja. 

\newpage{}

\begin{lstlisting}[caption=Algoritam praćenja sfere, label=alg:spheretracing]
pracenje_sfere(org, dir, max, epsilon) {
	// org - ishodiste zrake
	// dir - smjer zrake
	// max - maksimalan broj koraka
	// epsilon - vrlo mali broj
	pos = org
	for (count = 0; count < max; count++) {
		dist = sdf(pos) // funkcija udaljenosti
		if (dist < epsilon)
			break;
		pos += dir * dist
	}
	return pos
}
\end{lstlisting}

\section{Korišteni alati}
\label{podpog:koristeni_alati}

Primarni alat koji je korišten za ovaj rad je biblioteka \emph{WebGL2}. To je varijanta popularne grafičke biblioteke OpenGL koja je prilagođena za rad na internetskom pregledniku i jeziku \emph{JavaScript}. Mogućnost korištenja grafičkih funkcija na internetskom pregledniku ima mnoge prednosti: omogućuje grafičke vizualizacije unutar web stranica, izradu video igara koje su dostupne bez instaliravanja izvršne datoteke i generalni pristup procesorne moći grafićke kartice unutar \emph{JavaScript}. \emph{WebGL2} nije sasvim identičan OpenGL, jedna bitna razlika je što ne omogućuje izradu računskih sjenčara (\emph{engl. compute shaders}). Takvi sjenčari su pogodni za iscrtavanje temeljenom zrakama kao praćenje sfera, no moguće je izvesti takvo iscrtavanje koristeći sjenčare fragmenta.

Kao što je već spomenuto, programski jezik koji se koristi je \emph{JavaScript}. \emph{JavaScript} ima svojih prednosti što se tiće grafićkih aplikacija. Zajedno s \emph{HTML} i \emph{CSS} pruža jednostavan i fleksibilan način izrade korisničkog sučelja za upravljanje programom. Implementacija grafičkih sučelja u drugim jezicima kao \emph{C++} bi bila znatno teža i bilo bi potrebno koristiti datoteke kao na primjer \emph{Dear ImGui} \cite{imgui}, što ima manju fleksibilnost u dizajnu. Dodatno, web aplikacije su orijenirane na portabilan dizajn na način da jedna stranica može raditi na više vrsta uređaja. To uključuje i ovaj projekt: prikaz fraktala se može otvoriti na mobitelu, a sučelje se prilagođava na orijentaciju i veličinu ekrana. Naravno, kvaliteta i brzina iscrtavanja će ovisiti o grafičkom procesoru uređaja na kojem se izvodi.

Rezultat rada će se posluživati pomoću \emph{GitHub Pages}. To je praktično jer se \emph{Github} koristio za kontrolu verzija, pa se izvorni kod već nalazi na toj platformi. \emph{GitHub Pages} pruža statičko posluživanje, što je sasvim dovoljno za ovaj program.

\section{Postupak iscrtavanja}
\label{podpog:postupak_iscrtavanja}

Iscrtavanje je stvaranje slike virtualne scene koja se prikazuje na ekranu. Za scene s ciljem prikaza u stvarnom vremenu, frekvencija iscrtavanja mora biti dovoljno velika. Radi brzine, veliki dio postupka iscrtavanja je izveden hardverski i postoji definirani grafički protočni sustav kojeg se treba pridržavati da bi se nacrtao neki 3D model. U WebGL2 je obavezno definirati barem sjenčara geometrije (\emph{engl. vertex shader}) i sjenčara fragmenata (\emph{engl. fragment shader}). U ovome slučaju, modeli koje želimo prikazati se ne sastoje od trokuta i tehnički nije potrebna geometrijska faza. Bilo bi praktično koristiti računske sjenčare (\emph{engl. compute shaders}), no kao što je prije navedeno, \emph{WebGL2} ne omogućuje izradu računskih sjenčara. Stoga je postupak iscrtavanja izveden onime što imamo i što trebamo koristiti. U nastavku će biti objašnjeni postupak iscrtavanja u sastavljenom programu i biti će navedeni ključni objekti koji se koriste pri iscrtavanju. Slika \ref{slk:pipeline} prikazuje vizualno redoslijed grafičkih programa i objekte koji se opisuju.

Crtanje jednog okvira se izvodi u dva poziva na crtanje (\emph{engl. draw calls}). U svakom pozivu na crtanje se izvodi standardna grafička protočna struktura: sjenčar geometrije se iterira po točkama modela i sjenčar fragmenata se iterira po fragmentima. Jedini pravi model koji se koristi je kvadrat (dvoje trokuta) koji pokriva cijeli ekran. Geometrijski sjenčar samo prosljeđuje pozicije točaka modela kvadrata i ništa drugo. Model i geometrijski sjenčar su identična za oba poziva za crtanje. Gotovo sva logika koja se opisuje je u sjenčarima fragmenta.

Prvo iscrtavanje izvodi algoritam praćenje sfere i obavi sjenčanje. Za izračun boje piksela se uzima u obzir boja prošlog okvira. Pošto nije moguće iz istog spremnika čitati i pisati u ist vrijeme, napravljeni je par spremnika koji zajedno čine \emph{ping-pong buffer}. U jednom iscrtavanju, jedan spremnik služi za čitanje, a drugi za pisanje. Nakon svakog iscrtavanja se uloga tih spremnika zamijeni. Taj par spremnika koji omogućuje programu pristup informacijama o prošlom okviru se koristi uklanjanje alias artifakta, izvedbu praćenja zraka i opcionalan efekt fokusa kamere.

\newpage{}

Drugi poziv uzima sliku u međuspremniku, podesi joj boju i prikaže rezultat na \emph{canvas} element stranice. Korist dvoje poziva za crtanje je to što se u prvom pozivu (koji više računski intenzivan) može iscrtati slika manje rezolucije, a zatim se u drugom može skalirati međurezultat u rezoluciju za prikazivanje.

\section{Unos funkcije}
\label{podpog:unos_funkcije}

Funkcija daljine se unosi u tekstualni okvir u obliku izvornog programskog koda jezika \emph{GLSL}. \emph{GLSL} je progmramski jezik za sjenčare kojeg koristi \emph{WebGL2}. Nakon unosa se treba kliknuti gumb za kompajliranje da bi se primijenio kod. Prvo se izbriše postojeći program koji \emph{WebGL2} koristi za iscrtavanje, zatim se uneseni izvorni kod u obliku teksta spoji zajedno s izvornim kodom ostatka sjenčara fragmenta. Konačno se novi spojeni izvorni kod pošalje grafičkoj kartici, kompajlira se i stvori se novi program za iscrtavanje. Ako dođe do greške pri kompajliranju (kao na primjer sintaksna greška unesenog koda), funkcija daljine će se zamijeniti zadanom defaultnom funkcijom i ispisati će se poruka koja daje informaciju o greški radi jednostavnijeg popravljanja koda. 

Defaultna funkcija daljine koja se koristi kada se početno pokrene program ili kada dođe do greške u kompaliranju je navedena u algoritmu \ref{alg:sdf2}. To je ujedino i implementacija formule \ref{eq:sdf} iz podpoglavlja \ref{podpog:funkcije_udaljenosti} za oblik sfere. Sfera se nalazi u centru koordinatnog sustava i radius sfere poprima vrijednost varijable \emph{uniforms.custom\_b}. Program ima nekoliko uniformnih varijabla sjenčara koje su namjenje za korištenje u funkcijama daljine (u to je uključena \emph{uniforms.custom\_b}). Te varijable omogućuju podešavanje vrijednosti proizvoljnih brojeva u funkciji kroz sučelje tako da nije potrebno ponovno prevođenje koda nakon svake promijene.

\begin{lstlisting}[caption=Implementacija funkcije daljine za sferu, label=alg:sdf2]
float SDF(vec3 p) {
	float radius = uniforms.custom_b;
	return length(p) - radius;
}
\end{lstlisting}

\section{Načini prikaza}
\label{podpog:nacini_prikaza}

Prikaz tijela se ostvaruje u dva koraka: određivanje točke na površini tijela algoritmom praćenja sfere i pretravanje informacija o toj točki u boju na ekranu. Algoritam praćenja sfere je implementiran po pseudokodu \ref{alg:spheretracing} (koji je prije opisan) i kod rješenja se nalazi u algoritmu \ref{alg:spheretracing2}. Za algoritam se koriste dvoje struktura \emph{Ray} i \emph{Data}. \emph{Ray} opisuje zraku i koristi se kao ulaz u algoritam, a \emph{Data} služi za čuvanje podataka koje algoritam izračuna u jednom paketu. Pretvaranje podataka u boju je izveden u četiri načina i to su u ovom projektu nazvani "Načini prikaza". Slijedeća podpoglavlja služe za opis svakog načina prikaza.

\begin{lstlisting}[caption=Isječak koda za implementaciju praćenja sfere, label=alg:spheretracing2]
// struktura za opis zrake    
struct Ray {
    vec3 origin; // ishodiste zrake
    vec3 direction; // smjer zrake
};

// kontejner informacija za rezultat algoritma
struct Data {
    vec3 position; // tocka presjeka zrake i tijela
    bool collided; // je li zraka udarila tijelo
    vec3 normal; // normala povrsine u tocki presjeka
    float dist; // duljina puta kojem je zraka putovala
    int marches; // broj koraka kojih je zraka otputovala
};

// algoritam pracenja sfere
Data rayMarch(Ray ray) {
    Data data;
    data.position = ray.origin;
    data.collided = false;
    data.marches = 0;

    for (; data.marches < int(uniforms.max_marches); data.marches++) {
        float d = SDF(data.position);
        if (d < pow(2.0, -uniforms.detail)) {
            data.collided = true;
            break;
        }
        data.position += d * ray.direction;
    }

    // funkcija deriveNormal() je opisana u poglavlju "Normale"
    data.normal = deriveNormal(data.position, uniforms.normals_precision);
    data.dist = length(data.position - ray.origin); 
    return data;
}
\end{lstlisting}

\subsection{Normale}

Drugi naćin prikaza služi za prikaz vrijednosti normale (slika \ref{slk:normals}). Normala je jedinični vektor koji je usmjeren gdje površina gleda. Vrijednost normale se može izračunati derivacije funkcije udaljenosti za svaku os zasebno. 

\begin{figure}[htb]
	\centering
    \includegraphics[width=0.5\linewidth]{images/normals.png}
    \caption{Prikaz temeljen na vektoru normale površine}
    \label{slk:normals}
\end{figure}

Za aproksimaciju derivacije funkcije koristi se metoda središnjih razlika (\emph{engl. central difference method}, formula \ref{eq:cdm}). Funkcija se mora evaluirati dva puta za svaku os i kao parametar se koristi proizvoljan vrlo mali broj $dx$. Implementacija izračuna normale u točki zajedno s kodom za izračun boje je u algoritmu \ref{alg:normals} .

\begin{equation}
	\label{eq:cdm}
	f'(x) \approx \frac{f(x+\Delta x)-f(x-\Delta x)}{2 \Delta x}
\end{equation}

\begin{lstlisting}[caption=Isječak koda za način prikaza normala, label=alg:normals]
// varijabla "epsilon" je ekvivalentna "dx" iz gornje formule
vec3 deriveNormal(vec3 position, float epsilon) {
    vec3 normal;
    normal.x = SDF(position + vec3(epsilon, 0.0, 0.0)) 
        - SDF(position - vec3(epsilon, 0.0, 0.0));
    normal.y = SDF(position + vec3(0.0, epsilon, 0.0)) 
        - SDF(position - vec3(0.0, epsilon, 0.0));
    normal.z = SDF(position + vec3(0.0, 0.0, epsilon)) 
        - SDF(position - vec3(0.0, 0.0, epsilon));
    return normalize(normal  / (2.0 * epsilon));
}
// pretvaranje vektora normale u boju
void main() {
    ...
    Data data = rayMarch(camera_ray);
    float factor = 1.0 - (float(data.marches) / uniforms.max_marches);
    if (data.collided)
        pixel_color = vec3((data.normal * 0.5 + 0.5) * factor);
    else
        pixel_color = vec3(0.0);
    ...
}
\end{lstlisting}

\subsection{Globalno osvjetljenje}

Zadnji i četvrti način prikaza je globalno osvjetljenje (slika \ref{slk:pathtraced}). Koristi se pojednostavljena algoritam praćenja zrake zajedno s \emph{ping-pong} spremnikom. Kada zraka dođe do površine, odbije u nasumičnom smjeru i nastavi se odbijati dok ne prijeđe maksimalan broj odbijanja (rezultat je crna boja) ili otiđe u nebo (rezultat je boja neba u tom smjeru) (algoritam \ref{alg:pathtraced}). Konačni rezultat je prosijek svih boja koje su dobivene za dani piksel. Pošto je ovaj postupak računski intenzivan, zbog responzivnosti i optimizacije je prosijek računan između prošlog i trenutnog okvira koji se iscrtava. Time treba manje vremena za izračun pojedinog okvira, a slika se postepeno razvija u bolji razultat što se duže program ostavi da radi.

\begin{figure}[htb]
	\centering
    \includegraphics[width=0.5\linewidth]{images/pathtraced.png}
    \caption{Prikaz s globalnim osvjetljenjem}
    \label{slk:pathtraced}
\end{figure}

\begin{lstlisting}[caption=Isječak koda za implementaciju globalnog osvjetljenja, label=alg:pathtraced]
vec3 pathTrace(Ray camera_ray, inout uint seed) {
    Ray ray = camera_ray;
    for (int bounce_counter = 0; bounce_counter < int(uniforms.max_bounces); bounce_counter++) {
        Data data = rayMarch(ray);
        
        if (!data.collided && data.dist > 10.0)
            return skyValue(ray.direction);

        ray.direction = normalize(randomDirection(seed) + data.normal);
        ray.origin = data.position + ray.direction * uniforms.epsilon;
    }
    return vec3(0.0);
}

void main() {
    ...
    pixel_color = pathTrace(camera_ray, seed);
    // izracun prosijeka trenutne boje i boje proslog okvira
    output_color = vec4(previous_color.xyz * ((uniforms.temporal_counter - 1.0) / uniforms.temporal_counter) + pixel_color * (1.0 / uniforms.temporal_counter), 1.0);
    ...
}
\end{lstlisting}
